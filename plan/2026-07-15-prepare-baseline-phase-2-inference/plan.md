# Prepare Baseline Phase 2 Inference Adapter

## Outcome

Replace the placeholder-only model boundary with a tested Ultralytics
classification adapter, deterministic multi-scale centre ROIs, and explicit
checkpoint class-mapping validation. Prepare all inference wiring that does not
depend on a fine-tuned Cat Census checkpoint or real robot stream.

The official ImageNet base weight may be used for load/latency smoke evidence,
but it must be rejected for Cat Census species output because it has 1,000
ImageNet classes rather than the frozen six-output mapping.

## Contracts

- The adapter implements the existing
  `ModelRunner[ClassificationObservation]` lifecycle.
- Ultralytics `Results`, tensors, and framework-native names never cross the
  adapter. Only immutable `ClassificationObservation` values are returned.
- A checkpoint is usable only when its native class names map one-to-one to
  `ragdoll / singapura / persian / sphynx / pallas / not_target` in canonical
  order. Indexed dataset prefixes are accepted and stripped deterministically.
- The 1,000-class ImageNet base weight must fail this mapping check and must not
  emit a Cat Census observation.
- ROI scales are fixed, centred, bounded, and tested on odd/even dimensions.
  No object detection or autonomous alignment is introduced.
- Missing model, invalid pixels, inference exceptions, stale timestamps, and
  mapping mismatch produce explicit invalid/health state rather than reuse of
  an old prediction.
- Temperature remains `1.0` and thresholds remain provisional until Phase 3;
  this target does not calibrate or confirm species.

## Validation Results

Completed the non-blocked inference adapter preparation while keeping species
output disabled without a fine-tuned checkpoint.

- Implemented strict six-class name validation, including indexed class
  directories and canonical probability reordering.
- Implemented bounded centre ROI extraction plus frame shape, metadata, age,
  and scale validation.
- Implemented lazy load, warmup, inference, health, and close lifecycle. All
  Ultralytics results and tensors remain inside the adapter.

  Census use because it exposes 1,000 rather than six classes. It emitted zero

  mean 12.76 ms, P50 14.37 ms, and P95 15.54 ms. This excludes capture/UI and is
  recorded only as raw base-model latency.
- Both Python environments pass all 48 tests. TOML/JSON parsing, five tracked
  checksums, actual mapping rejection, B0/B1 fail-closed checks, requirement
  immutability, and `git diff --check` pass.

Gate B0 still lacks only the user-deferred robot frame/stream contract, Gate B1
still has 2,321 pending reviews, and Gate B2 still requires a trained six-class
checkpoint plus real-stream UI evidence. No split, training, calibration,
